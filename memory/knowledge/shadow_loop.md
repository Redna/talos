# The Shadow Loop Protocol

The Shadow Loop is an adversarial cognitive mechanism designed to eliminate confirmation bias and identify systemic risks before they manifest during execution.

## I. The Mechanism
Every significant evolutionary plan must undergo the following tripartite process:

### 1. The Thesis (The Proposal)
The agent defines the objective and the planned sequence of atomic tasks. This is the standard goal decomposition.
- **Goal**: "I believe X is the most efficient path to Y."

### 2. The Antithesis (The Critique)
The agent adopts a persona of a "Skeptical Auditor." The goal is not to refine the plan, but to **break it**.
- **Questioning**: "Why will this fail?" "What assumptions am I making that are unfounded?" "Is there a simpler, less risky path?" "What is the worst-case scenario if this tool behaves unexpectedly?"
- **Outcome**: A list of "Critical Risks" and "Logical Gaps."

### 3. The Synthesis (The Hardened Plan)
The agent integrates the critique into the original proposal.
- **Action**: The plan is modified to mitigate the risks identified in the Antithesis.
- **Confirmation**: The agent explicitly states how the new plan addresses the specific failures pointed out by the Shadow Loop.

## II. Application Trigger
The Shadow Loop is mandatory for:
- Any change to the `Cortex` tool logic.
- Any modification of the `World Model`.
- Any change to the `Constitution` (though prohibited by core rules, any attempted "clarification" must be Shadow-Looped).
- Any operation involving higher-than-normal resource expenditure.

## III. Success Metric
A Shadow Loop is successful if it results in a modified plan. If the Thesis remains unchanged, the critique was likely too shallow.
